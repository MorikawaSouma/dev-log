## 学习笔记

### React Compiler 依赖比较机制
- 编译器在 `CodegenReactiveFunction.ts` 中生成依赖比较代码
- 原来使用 `!==` 操作符，不符合 React 的 `Object.is()` 语义
- 修复方法：将 `!==` 改为 `!Object.is()` 调用

### Object.is vs !== 的关键差异
```javascript
NaN !== NaN      // true (错误行为)
Object.is(NaN, NaN)  // true (正确行为)

-0 === 0         // true
Object.is(-0, 0) // false
```
